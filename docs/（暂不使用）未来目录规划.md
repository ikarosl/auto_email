# 十三、后面让 AI 写代码时的目录约束

你可以在提示词里加这一段：

```text
请严格按照 docs/10-project-structure.md 中的目录结构开发。

当前阶段只允许实现以下后端模块：
- email-messages
- email-attachments
- customers
- inquiry-cases
- inquiry-messages

当前阶段只允许实现以下前端页面：
- email/EmailList.vue
- email/EmailDetail.vue
- inquiry/InquiryList.vue
- inquiry/InquiryDetail.vue
- customer/CustomerList.vue
- customer/CustomerDetail.vue

不要提前创建 RAG、报价、研发评审、自动发送相关业务代码。
如确实需要预留，只能在文档中说明，不要写实际实现。
```

这段很重要。
AI 写代码时不是不会写，是太会写。不给它边界，它会顺手给你造一座城市。

# 十四、结论

你的项目目录应该按这三个层次规划：

```text
apps       = 实际应用
packages   = 共享能力
docs       = 约束 AI 和开发人员的施工图
```

第一版重点目录是：

```text
backend/modules/email-messages
backend/modules/customers
backend/modules/inquiry-cases
backend/modules/inquiry-messages
admin-web/views/email
admin-web/views/inquiry
admin-web/views/customer
```

后续 AI、RAG、研发评审、报价交接都可以自然长出来，但现在不要让它们提前占地盘。先把主干长直，枝叶后面再开。
